# Project: error-union demo

This project demonstrates a custom Rust error-handling pattern with the following goals:

## Core Goals

1. **Flat error type (no nesting)**
   - There is only ONE public error type: `AppError`
   - No nested enums like `AppError::File(FileError::Parse(...))`
   - Tree height MUST always be 1

2. **Centralized error definition**
   - All error variants are defined in `error.rs`
   - Other modules MUST NOT define their own public error enums for propagation

3. **Automatic conversion using `?`**
   - All leaf errors (e.g. `ParseIntError`, `io::Error`) are converted into `AppError`
   - This is done via `From<T>` implementations generated by the derive macro

4. **Precise error location tracking**
   - Every error is wrapped in `Located<T>`
   - `Located<T>` captures the exact call site using `#[track_caller]`
   - The location MUST point to where `?` happens (NOT where unwrap/print happens)

5. **Zero nesting behavior**
   - If a function already returns `AppError`, then calling it with `?` MUST NOT re-wrap or change location
   - This ensures correct propagation

---

## Architecture

error.rs:
- Defines:
  - `AppError` (flat enum)
  - `Result<T>` alias

file1.rs:
- Produces parse errors

file2.rs:
- Calls file1 and produces IO errors

main.rs:
- Calls both and prints errors

---

## Key Rules

- ALWAYS return `crate::error::Result<T>`
- NEVER introduce new error wrapper types in modules
- ONLY leaf errors should be converted into `AppError`
- DO NOT wrap `AppError` again
- DO NOT create nested error structures

---

## Expected Behavior

### Case 1: Parse error
Location should point to:
file1.rs → line where `parse()?` happens

### Case 2: IO error
Location should point to:
file2.rs → line where `read_to_string()?` happens

### Case 3: file2 calling file1
- If file1 already returns `AppError`
- file2 MUST NOT change the error location

---

## What Codex Should Do

When modifying code:

- Preserve flat error design
- Preserve `Located<T>` wrapping
- Preserve `#[track_caller]` behavior
- Do not introduce nested error enums
- Do not duplicate Display/From implementations outside derive

==================== src/error.rs =======================

use error_union::{ErrorUnion, Located};

#[derive(Debug, ErrorUnion)]
pub enum AppError {
    Parse(Located<std::num::ParseIntError>),
    Io(Located<std::io::Error>),
}

pub type Result<T> = std::result::Result<T, AppError>;

========================================================
==================== src/file1.rs =======================

use crate::error::Result;

pub fn parse_number(input: &str) -> Result<u32> {
    let value = input.parse::<u32>()?;
    Ok(value)
}

========================================================
==================== src/file2.rs =======================

use crate::error::Result;
use crate::file1;

pub fn read_and_parse(path: &str) -> Result<u32> {
    let text = std::fs::read_to_string(path)?;
    let value = file1::parse_number(text.trim())?;
    Ok(value)
}

========================================================
==================== src/main.rs ========================

mod error;
mod file1;
mod file2;

use crate::error::Result;

fn run() -> Result<()> {
    let n1 = file1::parse_number("123")?;
    println!("parsed from file1: {n1}");

    let n2 = file2::read_and_parse("number.txt")?;
    println!("parsed from file2: {n2}");

    Ok(())
}

fn main() {
    if let Err(err) = run() {
        eprintln!("error: {err}");

        let mut source = std::error::Error::source(&err);
        while let Some(err) = source {
            eprintln!("caused by: {err}");
            source = err.source();
        }

        std::process::exit(1);
    }
}
